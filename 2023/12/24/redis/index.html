Redis 入门

The open source, in-memory data store used by millions of developers as a database, cache, streaming engine, and message broker.

Redis 简介

  • 基本数据类型
    • 字符串 String
    • 列表 List
    • 集合 Set
    • 有序集合 SortedSet
    • 哈希 Hash
  • 高级数据类型
    • 消息队列 stream
    • 地理空间 Geospatial
    • HyperLogLog
    • 位图 Bitmap
    • 位域 Bitfield
  • 使用方式
    • CLI (Command Line Interface)
    • API (Application Programming Interface)
    • GUl (Graphical User Interface)
  • Redis 优势
    • 性能极高
    • 数据类型丰富，单键值对最大支持512M大小的数据
    • 简单易用，支持所有主流编程语言
    • 支持数据持久化、主从复制、哨兵模式等高可用特性

事务

  • 在一次请求中执行多个命令
  • 并不能保证所有命令都执行成功，结果取决于命令
  • 但不会因为某个命令执行失败而导致其他命令不被执行
  • 事务执行命令序列中不会被新的请求插入
  • MULTI
  • EXEC/DISCARD
  • 与数据库的事务差别有些大

持久化

  • RDB (Redis Database)

    • dump.rdb 快照文件
    • 快照的时候是阻塞状态
    • 有一个子进程负责快照

  • AOF（Append Only File）

    • 追加文件
    • 同时写入内存和文件中

主从复制

  • 主节点负责写操作，异步发生数据到从节点

  • 从节点负责读操作

  • 通过命令或者配置文件

    • IP地址、端口号
    • 进程文件

  • redis-server redis-xxx.conf

  • redis-cli -p xxx

    • info replication

哨兵模式

  • 监控 Redis 集群各个节点的状态，自动的故障转移
  • 监控 - 通知 - 转移
    • 主节点挂了，从节点顶上
  • redis-sentinel sentinel.conf
  • 哨兵节点本身也可以是多个保证高可用
