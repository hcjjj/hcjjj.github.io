Windows 下安装 Linux 双系统

post @ 2019-08-03

总体流程

  1. 下载linux镜像文件
  2. 创建USB启动盘
  3. 磁盘分区
  4. 安装系统

介绍

linux选择

我选择的是 Manjaro ，官方介绍：

Manjaro是一款基于ArchLinux、对用户友好、全球排名第1的Linux发行版（排名数据源于DistroWatch，统计日期2018.03.02，时间段3个月）

它有3个无与伦比的优势：

  1. 滚动更新可以使软件保持最新；
  2. AUR软件仓库有着世界上最齐全的Linux软件（《一张列表展示ArchLinux系软件有多丰富——看哭百万Debian、RedHat系同学》）；
  3. 丰富的wiki和活跃的社区让所有问题都可以快速得到满意的答案。

Ubuntu、Deepin、Linux Mint 等其他发行版只要下载相应的镜像，安装过程大同小异

USB启动盘

我准备的是一个16G的U盘,用 Rufus 创建USB启动盘

启动盘(Startup Disk)，又称紧急启动盘(Emergency Startup Disk)或安装启动盘。它是写入了操作系统镜像文件的具有特殊功能的移动存储介质(u盘、光盘、移动硬盘以及早期的软盘)，主要用来在操作系统崩溃时进行修复或者重装系统。

具体流程

下载ISO文件

打开 Manjaro稳定版下载 选择下载适合自己的版本

下载Rufus

进入 http://rufus.ie ，右上角可选择语言

需要Windows 7以上的操作系统，无所谓32位还是64位，下载后开箱即用

制作USB启动盘

插入U盘，打开rufus，选择下载的iso文件，点击开始
一

写入时电脑不显示U盘，完成后U盘变为EFI启动盘

磁盘分区

演示的是win7系统，win10大同小异

磁盘分区就是为了将电脑的磁盘空间腾出来一部分给linux系统使用
右键 我的电脑 -> 管理 -> 磁盘管理 ，在你想安装linux 的盘 右键 压缩卷


这里我分出了120G的空间给linux，点压缩，这就分配好空间了

安装系统

进入BIOS选择U盘启动

进入系统后，双击桌面图标 Install Manjaro Linux ，一路设置

选择手动分区，在刚刚windows下压缩出来的空闲空间上分别创建分区

文件系统 挂载点 参考大小
ext4 / 20G
ext4 /home 剩下全部
swap 与内存一致
FAT32 boot/efi 100M

注意要点：

必须创建 boot/efi 分区，同时选择标记esp，并格式化为FAT32格式

安装完成后再次进入BIOS修改启动项为：

这就是之前创建的 boot/efi 分区，功能是在双系统启动时能够选择要进入的系统

XFCE版的Manjaro
你可以把一块磁盘专门用来装linux，这样就无需分出空间给linux，安装的时候选择那块磁盘直接安装，即无需手动创建linux分区

在 Android 上搭建Hexo博客

post @ 2019-07-27

前期准备

  • Termux : 一个Android下的高级的终端模拟器（酷安上可以下载）
  • Node.js : 一个基于 Chrome V8 引擎的 JavaScript 运行环境
  • Git : 一个开源的分布式版本控制系统
  • Hexo : 一款基于Node.js的静态博客框架

国光的教程 : Termux 高级终端安装使用配置

搭建步骤

安装 node.js、git

pkg update 更新软件源

建议先更换国内源：清华源

pkg install nodejs git 安装 nodejs、git

node --version 执行显示版本号

git --version 表示安装成功

安装 Hexo

npm install -g hexo-cli 安装 Hexo

hexo --version 查看版本号

初始化博客

你可以在任意位置创建文件夹来存放博客初始化的文件，这个文件夹就是博客的 根目录

mkdir hexoblog 创建博客根目录

cd hexoblog 打开根目录

hexo init 初始化hexo环境，可能需要几分钟，取决于网速

初始化成功后，可以查看到以下博客目录：

博客目录结构(左侧)

hexo s 启动hexo本地预览

我的博客根目录是在 Desktop/blog ，在根目录执行 hexo s 后，浏览器打开 http://localhost:4000 即可本地预览博客

部署到 Gitee

注册

打开 https://gitee.com/ 注册并登入

右上角 “加号” 新建仓库

⚠️ 仓库路径必须与Gitee账号 同名

⚠️ 是否开源设置为 公开

配置

添加git用户信息

git config --global user.name "xxx"

git config --global user.email "xxxxx@xx.com"

修改站点配置文件

复制仓库地址：

打开根目录下的_config.yml，在配置文件最后的位置，做如下修改：

repo: [复制下来的仓库地址]（注意：后有一个看个）

部署

hexo clean 删除本地缓存和静态文件

hexo g 生成新的博客静态文件

hexo d 提交生成的静态文件

如提示 ERROR Deployer not found: git

则 npm install --save hexo-deployer-git

根据提示输入 gitee账号和密码

推送完成后，进入 仓库 -> 点击服务 -> Gitee Pages 服务 -> 更新

更新完成后，即可通过 [你的gitee账号].gitee.io 访问所搭建的博客

每次修改博客内容提交到 gitee 后，都需要到管理界面更新 Gitee Pages 新的页面才会生效
